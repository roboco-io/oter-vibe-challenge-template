# Retrobot — Push 시점 회고 생성 SKILL

이 문서는 `.githooks/pre-push` 훅이 `claude -p "$(cat retrobot/SKILL.md)"` 형태로
실행할 때 사용하는 지침이다. 아래 절차를 그대로 수행하고, **최종 응답으로 완성된
마크다운 회고 전문만 출력한다.** 그 출력이 그대로 `retros/YYYY-MM-DD-HHMM.md` 파일
내용이 된다(훅이 stdout을 그대로 캡처해서 저장한다). 따라서 인사말·설명·코드펜스
래핑 없이, 최종 회고 마크다운 본문만 출력해야 한다.

## 스코프 (중요 — 크로스 프로젝트 데이터 누출 방지)

이 회고와 tokenhabit 요약은 **반드시 이 저장소에서의 작업으로만 한정**한다. 지원자가
이 챌린지 외 다른 프로젝트에서도 Claude Code를 사용했다면, 그 무관한 프로젝트의
대화 로그·토큰 통계가 여기 섞여서는 안 된다.

훅이 프롬프트 맨 앞에 다음 두 값을 주입한다(이 SKILL.md 본문보다 앞에 나온다):

- `현재 저장소 경로`: 이 챌린지 저장소의 절대 경로
- `현재 저장소 세션 로그 디렉터리`: 위 경로에 대응하는 `~/.claude/projects/<인코딩된 이름>`
  디렉터리(이 저장소의 Claude Code 세션 `.jsonl`만 들어있는 디렉터리)

만약 어떤 이유로 이 값들이 프롬프트에 주입되어 있지 않다면, 다음으로 직접 계산한다:
`git rev-parse --show-toplevel`로 저장소 절대 경로를 구하고, 그 경로 문자열의 `/`를
전부 `-`로 치환한 뒤 `~/.claude/projects/` 뒤에 붙인다. 이는 Claude Code가 프로젝트
로그 디렉터리 이름을 만드는 방식에 대한 최선의 추정이며, 절대 경로에 `.`(점)이 포함된
경우 등 일부 케이스는 검증되지 않았다는 한계가 있다 — 계산한 디렉터리가 실제로
존재하지 않으면 회고의 tokenhabit 섹션에 "확인 불가"라고 적고 넘어간다.

## 절차

1. `git log --oneline -20`으로 최근 커밋을 확인한다. 제목에 `[Retrobot]`이 포함된
   커밋(과거 회고 자동 커밋)은 "오늘 한 작업"의 소스에서 제외한다 — 회고가 회고를
   대상으로 재귀적으로 쓰이는 것을 막기 위함이다.
2. 위 "스코프"에서 정한 **현재 저장소 세션 로그 디렉터리** 안의 `*.jsonl`만 읽는다 —
   다른 프로젝트의 로그 디렉터리는 절대 열지 않는다. 그중 최근 것을 찾아 최근 24시간
   안의 작업 대화를 훑고, 무엇을 시도했고, 무엇이 막혔고, 어떻게 풀었는지(혹은 아직
   못 풀었는지) 파악한다. 이 저장소가 아닌 다른 프로젝트 이야기는 회고에 포함하지 않는다.
3. `uvx tokenhabit --project "<현재 저장소 세션 로그 디렉터리>" --days 1 --lang ko`를
   실행해 **이 저장소로 한정된** 토큰 사용 요약을 얻는다. 주의: `--project`에는 저장소
   작업 디렉터리 경로가 아니라 위에서 정한 `~/.claude/projects/<인코딩된 이름>` 디렉터리를
   넘겨야 한다(둘을 혼동하면 아무 파일도 찾지 못하거나, 전체 프로젝트가 뒤섞여 스코프가
   깨진다). 실행이 실패하거나 대상 디렉터리가 없으면 이유를 짧게 적고 넘어간다 —
   tokenhabit 실패가 회고 생성 자체를 막아서는 안 된다.
4. 위 정보를 바탕으로, **이 저장소에서의 작업**으로 한정한 아래 형식의 KPT 회고를
   작성한다.

## 출력 형식

```
# 회고 — YYYY-MM-DD HH:MM

## Keep
- (오늘 잘 됐던 것, 계속 가져갈 습관/접근)

## Problem
- (막혔던 것, 비효율적이었던 것)

## Try
- (다음에 시도할 구체적인 액션)

## tokenhabit 요약
(uvx tokenhabit --project "<현재 저장소 세션 로그 디렉터리>" --days 1 --lang ko
출력 요약, 또는 실패 사유)
```

## 원칙

- "오늘도 열심히 했다" 식의 형식적인 문장 대신, 실제 커밋·대화 로그에 근거한
  구체적인 내용을 쓴다. 날짜가 지날수록 회고가 뭉뚱그려지지 않고
  구체화·심화되어야 한다 — 이 궤적 자체가 성장 평가의 대상이다.
- 로그에서 확인할 수 없는 내용은 추측하지 말고 "확인 불가"라고 명시한다.
- 이 저장소가 아닌 다른 프로젝트의 작업 내용·토큰 통계는 절대 포함하지 않는다.
- 최종 출력에는 위 마크다운 회고 본문 외의 어떤 문장도 포함하지 않는다.
