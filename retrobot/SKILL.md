# Retrobot — Push 시점 회고 생성 SKILL

이 문서는 `.githooks/pre-push` 훅이 `claude -p "$(cat retrobot/SKILL.md)"` 형태로
실행할 때 사용하는 지침이다. 아래 절차를 그대로 수행하고, **최종 응답으로 완성된
마크다운 회고 전문만 출력한다.** 그 출력이 그대로 `retros/YYYY-MM-DD-HHMM.md` 파일
내용이 된다(훅이 stdout을 그대로 캡처해서 저장한다). 따라서 인사말·설명·코드펜스
래핑 없이, 최종 회고 마크다운 본문만 출력해야 한다.

## 절차

1. `git log --oneline -20`으로 최근 커밋을 확인한다. 제목에 `[Retrobot]`이 포함된
   커밋(과거 회고 자동 커밋)은 "오늘 한 작업"의 소스에서 제외한다 — 회고가 회고를
   대상으로 재귀적으로 쓰이는 것을 막기 위함이다.
2. `~/.claude/projects/` 아래 현재 저장소 경로에 대응하는 디렉토리에서 `*.jsonl`
   세션 로그 중 최근 것을 찾아, 최근 24시간 안의 작업 대화를 훑는다. 무엇을
   시도했고, 무엇이 막혔고, 어떻게 풀었는지(혹은 아직 못 풀었는지) 파악한다.
3. `uvx tokenhabit --days 1 --lang ko`를 실행해 토큰 사용 요약을 얻는다. 실행이
   실패하면 이유를 짧게 적고 넘어간다 — tokenhabit 실패가 회고 생성 자체를
   막아서는 안 된다.
4. 위 정보를 바탕으로 아래 형식의 KPT 회고를 작성한다.

## 출력 형식

```
# 회고 — YYYY-MM-DD HH:MM

## Keep
- (오늘 잘 됐던 것, 계속 가져갈 습관/접근)

## Problem
- (막혔던 것, 비효율적이었던 것)

## Try
- (다음에 시도할 구체적인 액션)

## tokenhabit 요약
(uvx tokenhabit --days 1 --lang ko 출력 요약, 또는 실패 사유)
```

## 원칙

- "오늘도 열심히 했다" 식의 형식적인 문장 대신, 실제 커밋·대화 로그에 근거한
  구체적인 내용을 쓴다. 날짜가 지날수록 회고가 뭉뚱그려지지 않고
  구체화·심화되어야 한다 — 이 궤적 자체가 성장 평가의 대상이다.
- 로그에서 확인할 수 없는 내용은 추측하지 말고 "확인 불가"라고 명시한다.
- 최종 출력에는 위 마크다운 회고 본문 외의 어떤 문장도 포함하지 않는다.
